The basics of testing in rust

I once read somewhere that code without tests is bad code. Although testing is a big part of code development we seem to learn it last so I am gonna flip that and start my rusty adventure with first learning how to test my code snippets. After going over the rust book once just to get a feel my study routine will start on chapter 11.

Topic 11.1 describes the basic tools used in testing code. All in all they are 3 primary macros and 1 attribute and return.

  • assert!
  • assert_eq!
  • assert_ne!
  • #[should_panic]
  • -> Result<(), String>

This chapter also describes ways to extend some of the above with a special message on failure so that those who follow you can have your insights into what went wrong.

Simple assert

pub fn greeting(name: &str) -> String {
    format!("Hello {}!", name)
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn greeting_contains_name() {
        let result = greeting("Carol");
        assert!(result.contains("Carol"));
    }
}

To really drum these tools into my head future chapter I will tackle will have one of each of these tests.
